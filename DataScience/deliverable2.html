Looking at the Data

The data source Baseball Savant is an MLB affiliate that collects baseball statistics for major league games. The initial data set here foul_balls includes 906 observations with 7 variables. The dataset was used originally to observe where the most dangerous foul balls land, which is discussed in depth in this story. Our launch_data set was also collected from this source. I also used Baseball Reference to gather data on standings for the 2016 MLB season. Just in my initial investigation, there appeared to be some NA data points, so well have to clean the data up. I also noticed that there is some inconsistent or inappropriate type formatting, so well have to clean that up as well.

Here is an overview of each dataset

foul_balls
  • matchup
    • The two teams that played. Formatted like Atlanta Braves vs New York Mets. In order to have a merge point for this set and the previous set, I am going to make a lookup table with the team names and their corresponding three character codes.
  • date
    • Date of the most foul-heavy day at each stadium. Formatted like yyyy-mm-dd.
  • hit_type
    • Fly, grounder, line drive, pop up or batter hit self. Formated as a string.
  • velocity
    • Recorded exit velocity of each hit blank if not provided. Formatted as a double
  • predicted_zone
    • The zone we predicted the foul ball would land in by gauging angles. Formatted as a factor.
  • camera_zone
    • The zone that the foul ball landed in, confirmed by footage. Formatted as a factor.
  • used_zone
    • The zone used for analysis. Formatted as a factor.
launch_data
  • date
    • Date of the game where this hit was recorded Formatted like yyyy-mm-dd.
  • field
    • Field where the game was played. Formatted as string.
  • home
    • Name of home teams 3 character abbreviation. Formatted as a string.
  • away
    • Name of away teams 3 character abbreviation. Formatted as a string.
  • velocity
    • Exit velocity of the ball hit. Formatted as a double.
  • angle
    • Angle of the ball hit in degrees. Formatted as a double.
standings
  • rank
    • Team rank overall. Formatted as a double
  • team
    • Team names 3 character abbreviation
  • games
    • Total number of games played. Formatted as a double.
  • wins
    • Total number of games won. Formatted as a double.
  • losses
    • Total number of games lost. Formatted as a double.
  • run_diff
    • Run differential (average runs scored - average runs allowed). Formatted as a double.

To finalize cleaning our data, we need to do some work on the matchup column of our foul_balls data

Specifically the matchup is formatted like Atlanta Braves vs New York Mets. We want to offer consistency with our other data sets, so we are going to split this matchup into home and away. Well need to make a lookup table though to convert the full team name into its 3 character abbreviation. I just created my own csv file for this in an editor, so well have to load that.

teams <- suppressMessages(read_csv("./teams.csv"))

Using regular expresions in R, we can split up the matchups into their home and away teams. We will want to grab everything in the string until we encounter vs or VS and store that as away team, since American convention holds that matchups are written as away vs home. We can throw away the vs or VS and then we can grab the second string and store it as the home team.

After a lot of trial an error to match the abbreviations to the full names in the foul_balls set, I finally realized the their was some unnoticed spaces at the front of the home teams and at the back of the away teams. This happened because of the way I used str_split(). I believe you could eat the whitespace, but I instead decided to add some spaces to the team names in the lookup table.

split <- str_split(foul_balls$matchup, "(vs|VS)")
mat <- matrix(unlist(split), ncol=2, byrow=TRUE)
matchup <- as.data.frame(mat)
foul_balls$home <- as.character(matchup$V2)
foul_balls$away <- as.character(matchup$V1)

teams$space_front <- paste("", teams$full)
teams$space_back <- paste(teams$full, "")
foul_balls$home <- teams[match(foul_balls$home, teams$space_front), "abb"]
foul_balls$away <- teams[match(foul_balls$away, teams$space_back), "abb"]

Predictions to Model

Based on the initial phase and the newfound data, some predictions I would like to model are:

  • Can you predict a teams record based on their run differential in the first month of the season?
    • If you could predict a teams record based on their run differential in the first month of the season, that would bring significant change to the climate of the first month of the season and I think there would be a difference in the trades occuring before the trade deadline in July. I will be exploring a win-loss percentage rather than an exact record at this time.
  • Do teams that hit more homeruns win more games?
    • If it was found that teams that hit more homeruns win more games, teams would probably want to draft more heavy hitters. Perhaps a look at which factors are significant in a teams record would be a good model.

Model Planning

First Ill work on a model for predicting a teams win-loss percentage based on their run differential in the first month of the season

To do this, I created a tibble run_differntial with the data I wanted. I filtered my data to only include games from the first month of the 2016 season. I then created a new column for the home teams run differential for that game as well as for the away team.

I then created another tibble named model. This is what we will be training and testing our model with.

run_differential <- tibble(date=game_data$date, 
                           home=game_data$home, 
                           away=game_data$away, 
                           h_score=game_data$h_score, 
                           v_score=game_data$v_score)
run_differential <- run_differential %>%
  filter(date >= "2016-04-03") %>%
  filter(date <= "2016-05-03")

run_differential <- run_differential %>%
  mutate(h_run_diff=(h_score-v_score), v_run_diff=(v_score-h_score))

# I had to use the following 2 lines to get the win-loss percentage as a vector rather than a dataframe.
# It seems when I extracted the match it stored it as a datafrane even though it was a list, which made
# accessing it a pain, so I decided to strip it down to a vector first
win_loss <- standings[match(teams$abb, standings$team), "win_loss_percentage"]
win_loss <- as.vector(win_loss$win_loss_percentage)

model <- tibble(team=teams$abb,
                run_diff=0,
                win_loss_percentage=win_loss
                )

# I realize that this is a very inefficient algorithm, but for my purpose of only analyzing less 
# than 400 x 30 observations, I deemed it okay. I'm sure there is a way to leverage the vectorization
# of R to do this much more efficiently.
for (x in 1:length(run_differential$date)) {
  for (t in 1: length(model$team)) {
    if (model$team[t] == run_differential$away[x]) {
      model$run_diff[t] <- model$run_diff[t] + run_differential$v_run_diff[x]
    }
    if (model$team[t] == run_differential$home[x]) {
      model$run_diff[t] <- model$run_diff[t] + run_differential$h_run_diff[x]
    }
  }
}

Now we are ready to create our model

set.seed(1) # set the seed to yield consistent values in the data partition
selection <- createDataPartition(model$win_loss_percentage, p=0.75, list=FALSE)

train <- model[selection, ]
test <- model[-selection, ]


train_model <- lm(data = train, formula=win_loss_percentage~run_diff)

predictions <- train_model %>% 
  predict(test)

In summarizing our model, we can see that the run differential in the first month of the season is a good predictor of a teams win-loss percentage. We can see this because our p-value is very small. Additionally we can predict that for every 1 run increase in the run differneital, we will have a 0.0015 point increase in the win-loss percentage of that team.

Summary

With the models build here, I was able to find that a teams run-differential in the first month of the season is a good predictor of a teams win-loss percentage at the end of the season. Clearly the predicition that homeruns would be telling of a teams wins was a dead end though. Some possible limitations to this model would be teams have the potential to start very strong in the season and die off I suppose. There are a lot of factors that would contribute to a teams success in a season; however, in the model here it was pretty compelling to see the relationship between the run differential and the teams win-loss percentage. Some ways to move forward with this model would be to train the model based on before and after the All-star break, by month, etc. Perhaps there are more significant time frames that would provide a more accurate result for a teams win-loss percentage. Something that also came to my attention is that since the model is based on a domain of teams, there is not a ton of data. There are only 30 teams in the MLB, so in using this model, I am essentially seeing how well I can predict an individual teams win-loss percentage based on other teams run-differentials and their win-loss percentages. I am optimistic since the p-value was so extremely low; however, the scarcity of data in this model might not hold up. In order to solidify the validity of this model I would need to cross-validate it based on several other seasons. Keep in mind though, that even though the model set seems small, it is built from a transformation of very large datasets. So in its small appearance, there is actually a lot of collected data within it.
